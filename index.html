iOS应用安全测试

Veröffentlicht am 2018-03-18

####0x01 不安全的数据存储
a.账号密码使用Plist或者NSUserDefaults明文储存在沙盒中。
检查方法：使用cycript勾住DamnVulnerableIOSApp 打印沙盒路径 然后把沙盒的内容拷贝到电脑。NSUserDefaults保存在Library/Perferences/目录下。
b.Core Data存储
Core Data会以sqlite文件存储在沙盒中，使用Navicat可以清晰的看到保存的数据。
c.重要数据存储在钥匙串中
在非越狱设备上数据储存在钥匙串中是相对安全的，但是在越狱设备上我们也可以很容易从钥匙串中拿到关键数据。Snoop可以检测钥匙串数据库文件上的读取写入等操作，我们也可以使用Keychain Dumper导出钥匙串中的文件数据。
d.当图片或者一些重要文件需要 保存在本地的时候，我们可以使用加密文件的方式增加黑客破解的难度。

####0x02 越狱检测
a.检测常见越狱文件路径 /Applications/Cydia.app /Library/MobileSubstrate/MobileSubstrate.dylib /bin/bash /usr/sbin/sshd /etc/apt
b.检测Cydia的URL scheme(cydia://)
c.检测SSH Daemon
d.检测是否有权限读写沙盒之外的文件。

####0x03 绕过登录验证
a.如果是一个方法检测账号密码是否正确，并且返回值是BOOL类型。直接使用lldb附加到进程断点到方法最后一行，然后直接修改x0寄存器的值为1。
b.如果在方法中验证了密码是否正确，就在汇编代码和逻辑图中分析发现有一个比较x8如果等于0就弹出密码错误弹框，所以直接下断点修改x8寄存器的值。

####0x04 寻找内存中的敏感信息
有一些敏感数据我们在存储到本地时会对它进行加密，但是当程序运行中，这些信息往往还没有被加密，所以应该立即将他们释放。
a.一些数据在程序运行中，我们可以使用cycript来调试找到内存中还没有被释放的数据。

####0x05 监测盗版
a.在项目可能会利用很多方法来监测App是否被盗版，通过查看汇编代码发现了一个cbz的指令，猜测通过hopper把指令改成cbnz即可绕过检测，但是由于试用版没法导出改过的二进制文件。

####0x06 URL Schems
a.在iOS9之前 我们可以用URL Schems打开任意一个app，不需要任何验证这是很不安全的，但是在iOS9开始，苹果加强了权限，只要在info.plist文件中配置了URL Schems白名单才可以调用canopenurl:或是openurl:。

####0x07 Webkit Caching
WebKit的缓存分为三种，分别是页面缓存Page Cache，内存缓存Memory Cache，磁盘缓存Disk Cache。

####0x08 攻击第三方库
我们的iOS App中一般会用到很多的第三方库，比如统计，IM，直播等。而这些SDK通常需要App key，App Secret来完成初始化注册。此时我们就可以通过动态调试或者hopper的反汇编很容易拿到这些SDK的秘钥。

####0x09 在Mac上测试App
我们需要拿到App的源代码，才可以用xcode在模拟器上面运行。
a.我们需要确保这两个目录下面没有储存程序的敏感数据。
iOS 8 - /Users/[username]/Library/Developer/CoreSimulator/Devices/[a unique number]/data/Containers/Data/Application/[another unique number]/
iOS 7 - /Users/[username]/Library/Application Support/iPhone Simulator/[ios version]/Applications/[a unique id]/
b.检查程序运行时的Log打印日志。开发者应当监测当前运行设备是模拟器还是真机，在真机绝不可以有日志输出。
c.我们可以使用xcode内置的lldb来断点调试方法，使用cycript附加到进程，也可以在模拟器安装 SSL证书来监测网络流量。

####0x0a 不安全的密码加密
有时当我们把密码加密储存到本地时，我们认为他是安全的，但其实并非如此。
a.我们可以在App输入密码时，程序会验证密码是否正确，我们一般应该从-(BOOL)textFieldShouldReturn:(UITextField *)textField.方法入手，来查看验证过程。通过hopper的分析我们可以清楚的看到密码加密使用的类，我们可以拿到本地加密的密码，加上查询相关资料，找到这个加密类的解密方法，并且破解他。

####0x0b iOS的反调试与破解
Ptrace是一个系统调用，防止一个进程挂载到app上。
a.当一个应用被调试的时候，会给进程设置一个标识（P_TRACED），我们可以通过检测该进程是否有设置这个标识来检测进程是否正在被调试。

      
#import <dlfcn.h>
#import <sys/types.h>
typedef int (*ptrace_ptr_t)(int _request, pid_t _pid, caddr_t _addr, int _data);
#if !defined(PT_DENY_ATTACH)
#define PT_DENY_ATTACH 31
#endif  // !defined(PT_DENY_ATTACH)

void disable_gdb() {
  void* handle = dlopen(0, RTLD_GLOBAL | RTLD_NOW);
  ptrace_ptr_t ptrace_ptr = dlsym(handle, "ptrace");
  ptrace_ptr(PT_DENY_ATTACH, 0, 0, 0);
  dlclose(handle);
}

b.关于绕过ptrace一般是使用lldb断点在进入ptrace函数时，直接return 0，即可绕过ptrace的检测。

####0x0c 常用工具介绍
a.Cycript:它是混合了objective-c与javascript语法的一个工具，让开发者在命令行下和应用交互，在运行时查看和修改应用。关于cycript详细内容：Cycript
b.LLDB:它是XCode内置的为我们开发者提供的调试工具,能够带给我们更丰富的流程控制和数据检测的调试功能。关于lldb详细内容：LLDB
c.IDA和Hopper：IDA是目前最棒的一个静态反编译软件。而Hopper类似于IDA，但是功能没有那么强大，支持静态分析反汇编，流程控制图，伪代码，以及二进制代码。
关于IDA详细内容：IDA
关于Hopper详细内容：Hopper
d.Snoop-it：这个工具也很强大，支持查看钥匙串中储存的数据，查看app文件的属性和方法，分析网络流量等。关于Snoop-it详细内容：Snoop-it
e.MonleyDev:支持非越狱调试，以及hook方法编写，动态调试，自动集成了Cycript，Reveal，ANYMethodLog，dump头文件等功能。关于MonkeyDev详细内容：MonkeyDev
f.ANYMethodLog 可以打印oc类中的任何方法，支持动态打印。关于ANYMethodLog详细内容：ANYMethodLog

Mac版本微信QQ防撤回

Veröffentlicht am 2018-02-06

本文需要用到的工具：

  • hopper disassembler
  • Mac版本微信QQ

我是一个刚刚接触逆向的iOS开发者，本着实践出真知的理念在网上搜索资料，开始自己动手研究。看完这篇文章您将实现Mac版本微信QQ的消息防撤回功能。

####一、Mac版本微信
1.首先从App Store下载微信安装。
2.找到应用程序中的微信，右击显示包内容，找到MacOS文件夹下面的WeChat，这就是微信的可执行文件。

3.把微信的可执行文件WeChat拷贝到桌面备用。
4.这时候要拿出我们的关键工具：hopper disassembler ，把桌面的WeChat直接拖到hopper里面进行编译。这个时候需要等待的时间可能会久一点。

5.下图是编译完成后的样子，右下角的红点working消失，表示编译完成。

6.然后在左侧的搜索栏直接查找关键函数 onRevokeMsg

7.这个时候开始修改汇编代码：鼠标点击函数中第一行的 rbp，然后按 Option + A，输入ret，并点击 Assembler and Go Next，此时窗口会跳到下一行，点击 ESC 退出即可。

8.此时点击 hift + Command + E 重新生成修改后的微信可执行文件。

9.把修改后的WeChat复制到 /Applications/WeChat.app/Contents/MacOS 替换原来的WeChat，并关闭微信重新打开，此时别人无法撤回你的消息。Well done!

####二、Mac版本QQ

1.准备工作同上。首先从App Store下载QQ安装，去应用程序里面找到QQ，右击显示包内容，找到Contents/MacOS/QQ，把QQ复制到桌面，然后把QQ拖进hopper编译。
2.左侧搜索handleRecallNotify函数，鼠标点击函数中第一行的 rbp，然后按 Option + A，输入ret，并点击 Assembler and Go Next，此时窗口会跳到下一行，点击 ESC 退出。最后点击 hift + Command + E 重新生成修改后的QQ可QQ执行文件。
3.把修改后的QQ复制到 /Applications/QQ.app/Contents/MacOS 替换原来的QQ。重新打开QQ。
